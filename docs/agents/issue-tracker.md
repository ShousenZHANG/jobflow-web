# Issue Tracker: GitHub

Issues and PRDs for this repo live in GitHub Issues for `ShousenZHANG/jobflow-web`.

Use the `gh` CLI for issue operations when available. If `gh` is unavailable, use the GitHub connector/API with the same issue workflow.

## Conventions

- Create issues for PRDs, implementation slices, triage notes, and agent-ready briefs.
- Read full issue bodies, comments, and labels before triage or implementation planning.
- Apply labels through the triage mapping in `docs/agents/triage-labels.md`.
- Do not close or modify parent issues when creating implementation slice issues.

## When a skill says "publish to the issue tracker"

Create a GitHub issue in `ShousenZHANG/jobflow-web`.

## When a skill says "fetch the relevant ticket"

Read the GitHub issue body, labels, and comments before acting.
