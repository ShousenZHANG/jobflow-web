## Why Joblit

Most job-search tools stop at listing or tracking. Joblit closes the entire loop:

| Stage | What Joblit Does | How |
|---|---|---|
| **Discover** | Batch-fetch roles from job boards | GitHub Actions + Python JobSpy pipeline |
| **Triage** | Fast scan, filter, status-track | Two-pane workspace with search and JD rendering |
| **Tailor** | Generate targeted resume and cover letter | AI prompt engine with versioned rule templates |
| **Apply** | Auto-fill any ATS application form | Chrome extension with self-learning knowledge base |
| **Export** | Production-ready PDF output | LaTeX compile pipeline with bilingual support |

The differentiator: the **Chrome Extension** learns from every application you submit, building a personal knowledge base that makes each subsequent fill faster and more accurate.

## Core Features

### 1. Job Intake Pipeline

- Create `FetchRun` tasks with configurable role categories, markets, and filters
- Dispatch GitHub Actions to run Python JobSpy fetcher across multiple job boards
- Import with dedupe on `userId + jobUrl` and tombstone filtering for permanently dismissed URLs

### 2. Jobs Workspace

- Two-pane review UI: fast scanning on the left, full JD with markdown rendering on the right
- Search, filter, pagination, and status transitions (`NEW` / `APPLIED` / `REJECTED`)

### 3. Resume Studio

- Multi-step master-resume editor: basics, summary, experience, projects, education, skills
- Bullet-level edits with markdown emphasis, preview, and per-locale profiles (EN / CN)

### 4. AI Tailoring Engine

- Versioned prompt rule templates with skill-pack delivery
- Prompt generation for resume and cover letter targets
- Strict schema validation, `promptMeta` consistency checks, evidence-grounded bullet additions
- Manual JSON import path: copy prompt to external model, import result back

### 5. PDF Export

- LaTeX-based resume and cover letter rendering with bilingual templates (EN / CN)
- Optional persistent storage via Vercel Blob

### 6. Batch Automation (Codex Workflow)

- Create `NEW`-scope batches from filtered jobs
- Atomic claim-and-complete loop via `/api/application-batches/:id/run-once`
- Progress tracking via batch summary API

## Chrome Extension

A Manifest V3 Chrome extension that auto-fills job application forms using your Joblit resume profile.

### ATS Adapters

Purpose-built field detection for major applicant tracking systems:

| ATS | Adapter | Coverage |
|---|---|---|
| Workday | `workday.ts` | Input fields, dropdowns, custom components |
| Greenhouse | `greenhouse.ts` | Standard application forms |
| Lever | `lever.ts` | Application and referral forms |
| iCIMS | `icims.ts` | Career portal forms |
| SuccessFactors | `successFactors.ts` | SAP-based application flows |
| Generic | `generic.ts` | Fallback for unknown ATS platforms |

### Self-Learning Knowledge Base

The extension improves with every application:

1. **Fill** - Auto-fill detected fields from your resume profile
2. **Review** - Floating widget shows what was filled; edit any value inline
3. **Learn** - Each correction auto-saves as a mapping rule (field selector + profile path)
4. **Recall** - Next time the same field appears, the knowledge base supplies the corrected value

Rule matching priority: user rules > exact form signature > same ATS + domain > same ATS cross-domain.

Rules are scoped at three levels (global / ATS-level / site-specific) and managed from the web dashboard.

### Key Technical Details

- **Shadow DOM widget** - Vanilla JS, no framework dependency in content script (~39 KB)
- **React-compatible input simulation** - Dispatches native events to work with React, Angular, Vue controlled inputs
- **Form signature** - djb2 hash of field structure for cross-session form matching
- **Offline queue** - Submissions queue locally and sync when back online
- **Bilingual UI** - English and Chinese via lightweight i18n system

## Codex Batch Workflow

1. Fetch jobs and keep target roles as `NEW`.
2. Create batch: `POST /api/application-batches` with scope `NEW`.
3. Loop: call `POST /api/application-batches/:id/run-once` (atomic claim + complete).
4. For each task: generate prompt, run external model, import result.
5. Track progress: `GET /api/application-batches/:id/summary`.

## Environment Variables

Create a `.env` file in the project root.

**Required**

| Variable | Purpose |
|---|---|
| `DATABASE_URL` | PostgreSQL connection string |
| `AUTH_SECRET` | NextAuth secret |
| `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET` | Google OAuth |
| `GITHUB_ID` / `GITHUB_SECRET` | GitHub OAuth |
| `IMPORT_SECRET` | Admin import API auth |
| `FETCH_RUN_SECRET` | Fetch worker API auth |
| `APP_ENC_KEY` | Extension token encryption (base64) |
| `LATEX_RENDER_URL` / `LATEX_RENDER_TOKEN` | LaTeX PDF service |

**Optional**

| Variable | Purpose |
|---|---|
| `GEMINI_API_KEY` / `GEMINI_MODEL` | AI provider |
| `BLOB_READ_WRITE_TOKEN` | Vercel Blob storage |
| `GITHUB_OWNER` / `GITHUB_REPO` / `GITHUB_TOKEN` | Fetch workflow dispatch |
| `JOBLIT_WEB_URL` | Public URL for extension callback |

## Deployment

- **Recommended**: Vercel + Neon (PostgreSQL)
- Configure all environment variables in the deployment platform
- Chrome extension is distributed as a `.zip` for sideloading or Chrome Web Store

## Troubleshooting

| Symptom | Fix |
|---|---|
| `GITHUB_DISPATCH_FAILED` | Verify workflow token permissions |
| Low import volume | Increase fetch breadth or relax role excludes |
| `PROMPT_META_MISMATCH` | Re-download skill pack and re-copy prompt |
| Extension fields not filling | Check token validity on the Extension page |
